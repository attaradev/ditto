# Code of Conduct

## Our standard

ditto is an engineering project for users and contributors who value clarity, safety, and respect.
By participating in this repository, issue tracker, and related discussions, you agree to:

- act professionally and assume good intent
- give technical feedback about the work, not about the person
- welcome questions from users and first-time contributors
- document decisions clearly when you disagree
- respect maintainers' time by reporting issues with enough detail to reproduce them

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, intimidation, or personal attacks
- discriminatory language or exclusionary behavior
- doxxing or sharing private information without consent
- bad-faith arguments, trolling, or repeated disruption
- publishing security issues publicly before responsible disclosure

## Scope

This code of conduct applies to:

- GitHub issues, pull requests, and discussions
- code review and maintainer feedback
- community spaces directly tied to this project

## Reporting

For conduct issues that should not be discussed publicly, email **<security@attara.dev>** with the
subject line `Code of Conduct`.

Include:

- what happened
- where it happened
- who was involved
- any screenshots, links, or context needed to investigate

## Enforcement

Project maintainers may remove comments, reject contributions, or limit participation when behavior
violates this policy. Responses depend on severity and pattern of behavior, with the goal of keeping
the project safe and constructive for users and contributors.
